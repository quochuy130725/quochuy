OOP Concepts

Object-Oriented Programming (OOP) là một mô hình lập trình dựa trên các đối tượng (objects), cho phép lập trình viên mô phỏng các thực thể trong thế giới thực. Các concepts chính của OOP bao gồm:

  1. Encapsulation (Đóng gói):
      Bảo vệ dữ liệu bên trong một đối tượng bằng cách ẩn thông tin và chỉ cung cấp các phương thức (methods) để truy cập.
      Code 
      
                class Car {
                    private String model; // Thuộc tính được đóng gói
                
                    public void setModel(String model) {
                        this.model = model;
                    }
                
                    public String getModel() {
                        return model;
                    }
                }
  2. Inheritance (Kế thừa):
      Cho phép một lớp (class) kế thừa thuộc tính và phương thức từ lớp khác.
      Code 
      
                class Animal {
                    void eat() {
                        System.out.println("This animal eats food.");
                    }
                }
                
                class Dog extends Animal {
                    void bark() {
                        System.out.println("This dog barks.");
                    }
                }
      Cách dùng
      Code 
      
                public class Main {
                    public static void main(String[] args) {
                        Dog myDog = new Dog();
                        myDog.eat(); // Output: This animal eats food.
                        myDog.bark(); // Output: This dog barks.
                    }
                }
  3. Polymorphism (Đa hình):
      Một phương thức hoặc đối tượng có thể biểu diễn nhiều hành vi khác nhau.
      Code 
      
              class Animal {
                  void sound() {
                      System.out.println("Animal makes sound");
                  }
              }
              
              class Cat extends Animal {
                  void sound() {
                      System.out.println("Cat meows");
                  }
              }
  4. Abstraction (Trừu tượng):
      Ẩn đi các chi tiết thực thi, chỉ hiển thị hành vi bên ngoài của đối tượng. Chi tiết hơn được trình bày ở phần 3.2.
