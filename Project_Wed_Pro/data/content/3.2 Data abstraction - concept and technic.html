Data Abstraction

Data Abstraction là một trong những nguyên tắc cốt lõi của OOP, tập trung vào việc hiển thị các tính năng quan trọng của một đối tượng và ẩn các chi tiết không cần thiết.

    Concept
    • Trừu tượng hóa là quá trình mô tả hành vi của đối tượng mà không quan tâm đến cách nó được triển khai.
    • Giúp giảm sự phức tạp và tăng tính bảo mật của chương trình.
    Techniques for Data Abstraction

    1. Abstract Class:

        Một lớp có thể chứa cả phương thức trừu tượng (không có thân hàm) và phương thức thông thường.
        Code 
        
                      abstract class Shape {
                          abstract void draw(); // Phương thức trừu tượng
                      }
                      
                      class Circle extends Shape {
                          void draw() {
                              System.out.println("Drawing a circle");
                          }
                      }

    2. Interface:

        Chỉ chứa các phương thức trừu tượng (Java 8 trở lên có thể có phương thức mặc định).
        Code 
        
                        interface Drawable {
                            void draw(); // Phương thức trừu tượng
                        }
                        
                        class Rectangle implements Drawable {
                            public void draw() {
                                System.out.println("Drawing a rectangle");
                            }
                        }
    Ưu điểm của Abstraction:
    • Tăng tính bảo mật: Ẩn các chi tiết nội bộ.
    • Giảm độ phức tạp: Tập trung vào hành vi chính.
    • Linh hoạt: Dễ dàng bảo trì và mở rộng chương trình.
